# Blipzo App v1.0.0 Release Notes

## Overview
- **Version:** v1.0.0
- **Release Date:** 2026-01-01
- **Summary:** v1.0.0 marks the initial release of the Blipzo application, a modern, offline-first personal finance tracker. This release delivers core functionality for tracking income and expenses, robust authentication, and seamless data synchronization, built on a scalable architecture.

## Features
- **Offline-First Architecture**: Full functionality without internet access; data automatically syncs when connectivity is restored.
- **Transaction Management**: 
    - Track Income and Expenses with detailed metadata.
    - "Today" view for quick daily insights.
    - "All Transactions" view for comprehensive history.
- **Smart Categorization**: Manage and organize finances with custom categories.
- **Authentication**: Secure user session management (Login/Register) utilizing JWT.
- **Theming System**: Native support for Light and Dark modes with a custom design system.
- **Settings & Profile**: Tabs for managing user profile, categories, and application preferences.

## Bug Fixes
- **Network Stability**: Resolved intermittent "Network Error" issues on the `/health` endpoint (Heisenbug fix).
- **UI Rendering**: Fixed z-index issues with dropdowns and alignment problems with toggle pills.
- **Connection Handling**: Improved error handling for unstable network conditions.

## Improvements / Refactors
- **Service Layer Architecture**: Refactored backend logic from Controllers to dedicated Services (`src/api/v1/services/`) for better separation of concerns and maintainability.
- **API Versioning**: Standardized all API endpoints to use `/api/v1/` prefix for future-proofing.
- **Settings UI**: Redesigned Settings screen to use a tabbed interface for better navigation and scalability.
- **Tailwind/Styling**: Migrated styling setup to a more robust production-ready configuration.

## Known Limitations
- **Sync Conflict Resolution**: Basic collision detection is in place; complex multi-device conflicts may require manual refresh.
- **Reporting**: Advanced visual charts and analytics are currently in development for v1.1.0.

## Setup / Run Instructions (high-level)
1.  **Clone & Install**:
    ```bash
    git clone <repo-url>
    cd blipzo-app
    npm install
    ```
2.  **Environment Setup**:
    - Create a `.env` file in the root.
    - Set `EXPO_PUBLIC_API_BASE_URL` to your backend URL.
3.  **Run Application**:
    ```bash
    # iOS
    npm run ios
    # Android
    npm run android
    # Start Development Server
    npm start
    ```

## Branching & Versioning Strategy
- **main**: Production-ready code. Deploys to production environment.
- **dev**: Active development branch. Deploys to staging for QA.
- **feature/***: Short-lived branches for new requirements (merged into `dev`).
- **hotfix/***: Urgent fixes for production issues (merged into `main` and `dev`).
- **Versioning**: Follows Semantic Versioning (Major.Minor.Patch).

## Notes
- Ensure the backend service is running and accessible before testing sync features.
- This release targets Expo SDK 54 and React Native 0.81.5.
